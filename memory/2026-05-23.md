## 邮件发送：QQ 邮箱 SMTP 编码 Bug

### 问题现象
用户收到附件正常的邮件，但 body 中文乱码。

### 根因分析
- 根本原因：**PowerShell → Node** 的中文字符串传递编码问题
- PowerShell 字符串编码经过 `process.argv` 传递给 Node 时已经乱码
- SMTP 服务器本身没问题，附件正常发送说明通道是通的

### 修复方案
- `smtp.js` 已支持 `--body-file` 选项（body 从 UTF-8 文件读取）
- 正确做法：body 内容写入临时 UTF-8 文件，用 `--body-file` 传给 Node
- **不要用** `--body` 传中文内容（会触发编码问题）

### 教训
- Windows 上通过 PowerShell 调 Node.js CLI 传中文参数，编码链脆弱
- 所有含非 ASCII 字符的 CLI 参数（subject、body）一律走 `--xxx-file` 文件方式

## 邮件发送 Bug 深度分析（第二次修复）

### 现象
第一封邮件 body 乱码，且附件虽然发送成功但读取不到（被邮件客户端隐藏）。

### 根因：沙箱路径限制
QClaw 的 Node 进程运行在**文件系统沙箱**里：
- workspace 路径 `C:\Users\lightningwork\.qclaw\workspace-ua58rsb93veqtxl7\` → 沙箱直接拒绝，Access Denied
- Downloads 和 Documents → 可访问
- `.env` 的 `ALLOWED_READ_DIRS` 只是逻辑校验层，更底层的沙箱先于它拒绝

### 附件看不到的第二层原因
附件是通过 base64 编码后内嵌到 MIME 里的，所以发送过程本身没问题（不依赖读取）。
但因为邮件内容乱码，邮件客户端解析 MIME 结构异常，导致附件显示不出来。

### 最终修复
1. 附件必须放 `Downloads` 或 `Documents`，不要放 workspace
2. 中文 body 内容用 `--body-file` 传文件，不走 argv 参数
3. 用 PowerShell 脚本 `send_email.ps1` 封装调用（文件路径用 Downloads）

```
sumpool_demo_executed.ipynb + sumpool_demo.ipynb 已成功发送到 16414556@qq.com

### 工程经验
- **不要假设 CLI 工具的文件路径访问权限**：沙箱可能在应用层之前就拒绝
- **发邮件带附件的最佳实践**：文件放 Downloads，通过 `--attach` 传递绝对路径
- **PowerShell 调用 Node.js 含中文参数的编码处理**：统一走文件方式，不走 argv

### 关键发现

**MCP Host 问题（重要踩坑）**
- mac/Linux 下 MCP_HOST 默认连 `localhost:18888`，但 WSL2 环境下 localhost 指向 WSL 自身而非 Windows 主机
- 解决方案：PowerShell 脚本里用 `http://host.docker.internal:18888` 或直接用 `mcporter call`
- 腾讯文档 MCP 工具（`qclaw_tdoc_mcp_call`）走的是独立 HTTP 端口，实测 Windows PowerShell 可直接调用

**腾讯文档文件导入标准流程（两步走）**
1. `manage.pre_import` → 获取 COS 上传链接（PUT URL + file_key + task_id）
2. 本地上传文件到 COS 后 → `manage.async_import` 触发导入
3. `manage.import_progress` 轮询 → progress=100 即完成

### exec 工具编码问题（重要发现）
- exec 工具本身跑在 PowerShell 5.x 进程下（从 ANSI 颜色码格式可见）
- **但可以用 `pwsh -Command "..."` 显式调用 PowerShell 7 子进程**
- 实测：`pwsh -NoProfile -Command "Write-Host '中文正常'; [Console]::OutputEncoding.EncodingName"` → UTF-8 无乱码 ✅

### 实用结论
以后 exec 里涉及中文参数的命令，用 `pwsh -Command "..."` 包装：
```powershell
pwsh -NoProfile -Command "your command with 中文参数"
```
